# 改进点 4：可配置快捷键与冲突检测

## 背景/问题
快捷键目前写死在 `src/ui/interaction/shortcuts.rs` 与运行时派发逻辑中，用户无法根据不同键盘布局或习惯调整。

## 目标
- 允许通过配置文件自定义快捷键。
- 启动时检测冲突并给出明确提示。

## 方案概要
1. 在配置中新增 `keymap` 字段，结构为 `action -> key`。
2. 启动时加载并合并默认快捷键；未配置的 action 继续使用默认值。
3. 解析后检查重复按键，冲突时展示弹窗并回退到默认值。
4. 帮助页（F3）展示最终生效的快捷键。

## 涉及模块
- 快捷键声明：`src/ui/interaction/shortcuts.rs`
- 按键解析与派发：`src/ui/runtime_dispatch/key_helpers/*`
- 配置：`src/config.rs`

## 验收标准
- 自定义 keymap 生效且帮助页同步更新。
- 冲突时给出清晰提示并保证功能可用。

## 非目标
- 不支持运行时热更新，仅在启动时加载。
